# CODEBUDDY.md — 炒股基本面学习项目

## 项目概述

这是一个从零构建的跨市场股票投资学习项目，专注于**基本面分析**。覆盖 A 股、港股、美股三大市场，目标是通过系统化的知识整理逐步建立完整的投资分析框架。

## 技术约定

- **颜色规则**：遵循 A 股习惯 — 涨（红色）、跌（绿色），与美股绿涨红跌相反
- **货币符号**：默认使用 ¥（人民币），港股辅以 HK$，美股辅以 $
- **语言**：所有内容使用简体中文，界面文字亦为简体中文
- **文件格式**：知识内容以 Markdown（.md）为主，数据表可辅以 CSV/XLSX
- **金融数据**：核心财务数据（营收/净利润/增速）必须使用 `stock-analyzer`（东方财富）或 `neodata-financial-search` 技能获取最新准确数据，不得仅依赖 WebSearch 得到的历史数据
## 图表 / 插画策略（强制）

**所有可视化内容必须使用实际图片文件，禁止纯文本/ASCII/代码块模拟图表。**

| 场景 | 用什么 | 示例 |
|------|--------|------|
| 数据图表（柱状图、折线图、趋势图） | **SVG 矢量图**，存到 `assets/` 目录，Markdown 用 `![](./assets/xxx.svg)` 引用 | 市场规模趋势、营收对比 |
| 结构化图表（产业链图、流程图、竞争格局图） | **SVG 矢量图**，同上 | 产业链全景图、公司关系图 |
| 插画/知识卡片/行业概览 | **infographic-maker** 技能，生成手绘卡通风格，`assets/` 目录 | 行业入门卡片、关键概念图解 |
| 数据对比表、清单 | **Markdown 表格** | 各章节中的对比表 |

> 资产目录：`01-板块研究/{行业}/assets/`，图表文件与文档同级管理。

## 协作指引

### 对 AI 助手的期望
1. **数据优先**：涉及股票数据时，优先使用可用的金融数据技能获取实时/历史数据，而非依赖训练数据
2. **知识归档**：每完成一份研究报告或学习笔记，应归档到对应目录，格式统一为 Markdown
3. **Git 同步**：每完成一个完整模块（如某个行业的产业链、某家公司概况），自动 commit 并 push 到 GitHub
4. **中文输出**：所有回复和生成内容使用简体中文
5. **A 股颜色**：生成图表或表格时，涨用红色、跌用绿色

### Commit 规范
- 格式：`模块名: 简短描述`
- 示例：`板块研究: 添加半导体行业研究`

### 审查规则（强制）
**每次 commit + push 前，必须先启动审查 Agent**，对本次修改进行四项检查：
1. **准确性**：数据、名称、引用是否有误
2. **完整性**：文件间引用是否一致、是否有遗漏
3. **一致性**：目录结构、命名规范、文档格式是否符合本文件约定
4. **时效性**：财务数据、市场数据是否已更新到最新可获得的时间节点（年报/季报发布后及时刷新），是否存在明显过时的数据

审查通过后再 commit，不通过则修正后重新审查。

### 常用工作流
1. 用户提出某个行业想学习 → 获取数据 → 整理 Markdown → 归档到 `01-板块研究/{行业}/{市场}/` → 审查 Agent 检查 → commit + push → 同步到乐享知识库「stock」文件夹
2. 学习过程中发现需要新模块 → 讨论后增量扩展，不预设空目录
